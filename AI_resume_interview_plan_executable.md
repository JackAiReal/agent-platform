# AI简历优化 + AI面试助手项目落地方案（可执行版本）

## 一、项目定位

### 1. 项目名称（可暂定）
- OfferBoost
- 求职加速器
- AI求职教练

### 2. 项目目标
打造一个面向应届生、转行者、初中级求职者的 AI 求职工具网站，核心目标不是“生成内容”，而是**帮助用户提升拿到面试和 offer 的概率**。

### 3. 阶段性目标
- **30天内上线 MVP**
- **45天内拿到首批种子用户（100~300人）**
- **60天内验证付费模型，拿到第一笔稳定收入**

### 4. 核心价值主张
一句话：**让用户用更少时间，做出更强简历，并完成更像真的面试训练。**

---

## 二、目标用户与痛点

### 1. 目标用户
1. **应届生**
   - 没有完整简历经验
   - 不知道怎么写项目经历
   - 不知道面试官会问什么

2. **转行人群**
   - 旧经历和新岗位不匹配
   - 不会提炼 transferable skills
   - 容易在面试中被追问卡住

3. **初中级职场人**
   - 想跳槽拿更高薪资
   - 想投递更匹配的岗位
   - 需要更精准的岗位定制化简历

### 2. 核心痛点
- 不会根据岗位 JD 改简历
- 简历描述空、弱、泛
- 不知道 ATS / HR 喜欢什么关键词
- 面试前不会准备高频题和追问题
- 缺少低成本、高频训练环境

## 三、产品方案

## 1. MVP阶段：必须做的功能

### 功能A：AI简历优化
**目标：帮助用户从“普通简历”快速升级成“可投递简历”**

#### MVP功能清单
1. **上传简历**
   - 支持 PDF / DOCX / 文本粘贴
2. **输入目标岗位 JD**
   - 用户粘贴招聘描述
3. **AI 分析岗位要求**
   - 提取核心技能、关键词、职责、加分项
4. **AI 简历优化**
   - 按岗位重写自我介绍
   - 优化项目经历 bullet points
   - 补全可量化表达
   - 增强关键词匹配度
5. **生成优化报告**
   - 原文 vs 优化后对比
   - 缺失技能提示
   - ATS 关键词匹配评分
6. **导出结果**
   - 导出 Markdown / PDF / 文本

#### 结果页必须展示
- 岗位匹配度评分
- 问题项清单
- 优化后的简历内容
- 建议补充的项目/经历关键词

---

### 功能B：AI面试助手
**目标：让用户在正式面试前先练一轮，降低紧张和卡壳概率**

#### MVP功能清单
1. **选择岗位 / 输入 JD**
2. **AI生成面试题**
   - 自我介绍类
   - 项目经历类
   - 技术 / 业务问题
   - 行为面试题
3. **用户输入回答**
   - MVP阶段先做文本回答
4. **AI点评**
   - 回答是否清晰
   - 回答是否贴岗位
   - 是否缺少结果导向
   - 是否有表达问题
5. **AI追问**
   - 模拟真实面试官连续追问
6. **生成面试反馈报告**
   - 总体得分
   - 优点
   - 薄弱点
   - 推荐改进版本

#### MVP先不做
- 实时语音识别
- AI语音面试官
- 视频分析
- 超复杂题库管理后台

这些可以放到第二阶段，避免首版过重。

---

### 功能C：基础商业化功能
1. 用户登录/注册
2. 使用次数统计
3. 免费试用次数限制
4. 会员订阅 / 按次购买
5. 订单记录
6. 简单后台查看用户与订单数据

## 四、用户使用流程

### 路径1：简历优化
1. 用户注册登录
2. 上传简历
3. 粘贴目标岗位 JD
4. 系统解析 JD
5. AI 输出优化建议 + 改写版内容
6. 用户手动微调
7. 导出简历
8. 引导进入“模拟面试”流程

### 路径2：面试训练
1. 用户选择岗位 / 复用刚才的 JD
2. AI 生成 10~15 道面试题
3. 用户逐题回答
4. AI 点评并追问
5. 最终生成面试报告
6. 引导用户购买更多次数 / 开通会员

### 核心漏斗
- 访问首页
- 免费体验一次简历优化
- 看到“结果差距”
- 触发付费
- 再引导去面试助手提高客单价

最稳妥的方式：
- 免费体验：1次基础简历评分
- 付费包1：19.9 元（1次深度简历优化）
- 付费包2：39.9 元（简历优化 + 1次模拟面试）

### 3. 为什么先这样定价
- 降低首次付费门槛
- 让用户先感受到“结果差异”
- 方便后续根据转化率迭代定价

## 六、技术方案

## 1. 推荐技术栈（更适合快速上线）
- **前端**：Next.js 14 + TypeScript + Tailwind CSS
- **后端**：Next.js API Route / Server Actions
- **数据库**：PostgreSQL（推荐 Supabase 托管）
- **ORM**：Prisma
- **鉴权**：NextAuth 或 Supabase Auth
- **AI模型**：OpenAI 兼容接口（优先低成本模型做主流程，高质量模型做高级功能）
- **文件存储**：Supabase Storage / S3
- **支付**：Stripe（海外）/ 国内可先用手动收款或第三方支付聚合
- **埋点分析**：PostHog / Umami
- **错误监控**：Sentry
- **部署**：Vercel + Supabase

## 2. 为什么不建议首版单独拆 Node 服务
首版重点是验证产品，不是做微服务架构。
用 Next.js 全栈能更快上线、维护更轻、成本更低。
等业务跑起来，再考虑拆分独立服务。

## 九、AI能力设计

## 1. 简历优化 Prompt 逻辑
输入：
- 原始简历
- 目标岗位 JD
- 用户目标（实习/校招/社招/转行）

输出：
1. 岗位关键词提取
2. 简历问题诊断
3. 项目经历优化建议
4. 改写后的简历内容
5. ATS 匹配建议

### 关键要求
- 禁止虚构经历
- 可以优化表达，但不能替用户造假
- 优先保留真实经历，再强化结果和量化表述

## 2. 面试助手 Prompt 逻辑
输入：
- 岗位 JD
- 简历内容
- 用户回答

输出：
1. 面试题
2. 面试官追问
3. 回答评分
4. 问题点评
5. 更优示范回答

### 评分维度
- 相关性
- 结构清晰度
- 结果导向
- 真实性
- 表达流畅度

## 十四、风险与解决方案

### 风险1：AI输出空泛，用户觉得没价值
**解决：**
- 强化 JD 定制化能力
- 结果页显示“修改前后对比”
- 强调具体、量化、岗位匹配

### 风险2：成本过高
**解决：**
- 免费功能用低成本模型
- 深度报告再调用高质量模型
- 限制免费次数

### 风险3：转化低
**解决：**
- 先卖低价单次产品
- 在结果页制造清晰价值感
- 优化付费节点与文案

### 风险4：功能做太多，30天做不完
**解决：**
- 只做“简历优化 + 文本面试模拟 + 支付”
- 语音、视频、复杂后台全部后置

## 十七、结论
这不是一个“做 AI 功能展示页”的项目，而是一个**围绕求职结果的垂直工具产品**。

真正的关键不是模型多强，而是三件事：
1. **结果要看得见**：修改前后差异明显
2. **场景要够窄**：先打透求职这一个场景
3. **上线要够快**：先卖，再打磨

如果按这个版本执行，最合理的路径是：
**先做能付费的 MVP -> 拿到第一批用户 -> 根据反馈迭代功能 -> 再扩展到语音面试、行业模板和企业合作。**
