## 1. 项目概述

### 1.1 项目背景
当前求职市场中，应届生、转行者、初中级跳槽人群普遍存在以下问题：
- 不会根据岗位 JD 定制简历
- 简历表述弱、空泛、缺少结果导向
- 不清楚 ATS / HR 看重哪些关键词
- 面试前缺少高频、低成本、可重复训练场景
- 很难把“真实经历”转化成“可被录用的表达”

市面上很多 AI 工具只能“生成文本”，但不能真正围绕求职结果提供帮助。本项目的目标不是做一个泛 AI 工具，而是做一个围绕**拿到面试、提高 offer 概率**的垂直求职产品。

### 1.2 项目目标
打造一个面向求职用户的 AI 工具网站，提供两项核心能力：
1. **AI 简历优化**：根据目标岗位 JD，对简历进行诊断、重写和优化建议输出
2. **AI 面试助手**：生成岗位相关面试题，模拟追问，并给出回答点评与优化建议

### 1.3 阶段目标
- **30 天上线 MVP**
- **45 天获取首批 100~300 位种子用户**
- **60 天验证付费模型并产生第一笔稳定收入**

### 1.4 一句话定位
**帮用户做出更匹配岗位的简历，并在正式面试前完成高质量训练。**

## 2. 用户与场景

### 2.1 目标用户

#### 用户类型 A：应届生
特征：
- 简历项目经历弱
- 缺乏面试经验
- 不知道如何表达校园项目 / 实习经历

核心需求：
- 快速生成一份像样、可投递的简历
- 面试前获得高频问题训练

#### 用户类型 B：转行用户
特征：
- 原有经历和目标岗位匹配度低
- 不会提炼可迁移能力
- 容易在面试中被问倒

核心需求：
- 重构简历叙事
- 学会把旧经历包装成岗位需要的能力

#### 用户类型 C：初中级跳槽用户
特征：
- 想提升薪资或换更好的平台
- 需要更精准匹配不同岗位 JD
- 对面试题准备缺少系统性

核心需求：
- 快速做岗位定制化简历
- 获得针对性面试演练

### 2.2 核心使用场景
1. 用户看到某个目标岗位，想判断自己的简历是否匹配
2. 用户投递前，希望快速改出一版更强简历
3. 用户拿到面试机会，想提前练习高频题和追问
4. 用户面试后复盘，希望知道自己表达哪里弱

### 2.3 核心痛点
- 简历无法和 JD 一一对应
- 项目经历写不出亮点
- 回答面试题时缺少结构和结果感
- 不知道自己回答是否“像面试官会接受的答案”
- 缺少持续练习和低成本反馈

## 4. 产品核心价值

- 提高面试准备效率

## 5. 功能需求

## 5.1 功能模块 A：AI 简历优化

### 5.1.1 功能目标
帮助用户把原始简历升级为更适合目标岗位投递的版本。

### 5.1.2 用户流程
1. 用户进入简历优化页
2. 上传简历文件或直接粘贴简历文本
3. 粘贴目标岗位 JD
4. 系统提取 JD 核心关键词与岗位要求
5. AI 对原始简历进行问题诊断
6. AI 输出优化版内容与修改建议
7. 用户查看对比结果
8. 用户导出 / 复制优化内容
9. 系统引导进入模拟面试模块

### 5.1.3 输入项
- 简历原文（PDF / DOCX / 文本）
- 岗位 JD
- 求职阶段（可选：实习 / 校招 / 社招 / 转行）
- 目标岗位名称（可选）

### 5.1.4 输出项
- JD 关键词提取
- 简历问题诊断列表
- ATS 关键词匹配度评分
- 分模块优化建议
- 优化后的简历文本
- 建议补充的技能 / 项目描述方向

### 5.1.5 页面展示要求
结果页需展示：
1. **匹配评分**
2. **主要问题项**
3. **修改前 / 修改后对比**
4. **优化后的完整内容**
5. **建议补强项**
6. **CTA：继续做模拟面试**

### 5.1.6 验收标准
- 用户可成功上传或粘贴简历
- 用户可输入 JD 并完成分析
- 系统可返回结构化优化结果
- 页面可稳定展示对比信息
- 用户可复制或导出结果

---

## 5.2 功能模块 B：AI 面试助手

### 5.2.1 功能目标
帮助用户围绕目标岗位进行一轮文本版模拟面试，并获得具体反馈。

### 5.2.2 用户流程
1. 用户进入面试助手页
2. 输入或复用岗位 JD
3. 系统生成一组岗位相关问题
4. 用户逐题输入回答
5. AI 针对每题给出点评与追问
6. 完成整轮后生成面试报告
7. 用户查看得分、问题总结、推荐答案

### 5.2.3 题目类型
- 自我介绍题
- 项目经历题
- 专业能力 / 技术问题
- 行为面试题
- 追问题

### 5.2.4 输出项
- 问题列表
- 每题评分
- 每题点评
- 追问内容
- 推荐优化回答
- 总体总结报告

### 5.2.5 评分维度
- 岗位相关性
- 结构清晰度
- 结果导向
- 真实性
- 表达完整度

### 5.2.6 验收标准
- 可生成至少 10 道岗位相关问题
- 用户可逐题作答
- AI 可给出点评和追问
- 最终能输出结构化报告

---

### 5.3.2 功能点

## 9. AI 能力设计

## 9.1 简历优化 AI 逻辑
### 输入
- 简历内容
- JD 内容
- 用户求职阶段

### 输出结构
1. 岗位关键词提取
2. 简历问题诊断
3. 逐段优化建议
4. 改写后的简历文案
5. 匹配提升建议

### 规则约束
- 不允许虚构经历
- 不允许编造项目成果
- 可增强表达，但必须围绕原始经历
- 优先把描述改成“动作 + 方法 + 结果”的结构

4. 追问
5. 更优示范回答

### 规则约束
- 追问要贴近真实面试场景
- 点评必须具体，避免空泛表述
- 建议回答应保持真实性，不鼓励造假

## 11. 技术方案

### 11.1 推荐技术栈
- 前端：Next.js 14 + TypeScript + Tailwind CSS
- 后端：Next.js API Route / Server Actions
- 数据库：PostgreSQL（Supabase）
- ORM：Prisma
- 鉴权：NextAuth 或 Supabase Auth
- 文件存储：Supabase Storage / S3
- AI 接口：OpenAI 兼容 API
- 部署：Vercel
- 埋点：PostHog / Umami
- 错误监控：Sentry

### 11.2 技术原则
- 首版以快上线为优先，不拆复杂微服务
- 优先保证主链路稳定：上传 → 分析 → 结果 → 付费
- 模型调用需区分免费流程和付费流程成本

## 14. 风险与应对

### 风险 1：AI 输出太空泛
应对：
- 强化 JD 定制化
- 强展示“前后对比”
- 输出必须结构化

### 风险 2：模型成本过高
应对：
- 免费用户采用低成本策略
- 付费功能调用更高质量模型
- 控制免费额度

### 风险 3：30 天做不完
应对：
- 严格限制 MVP 功能
- 后置语音和视频能力
- 只做最核心漏斗

### 风险 4：用户觉得结果不可信
应对：
- 强调真实经历优化，不造假
- 给出解释型建议，而非只给结果
- 允许用户手动调整与确认
